## Dead Code False Positives

Documented in README as expected:

- **Orval-generated exports**: `getGetRestaurantsUrl`, `getRestaurants`, query key/option functions, type aliases for response variants — all generated by Orval from the OpenAPI spec. The hooks (`useGetRestaurants`, etc.) are what the app uses; the helper functions are internal Orval exports.

- **shadcn/ui component exports**: `badgeVariants`, `buttonVariants`, `CardFooter`, `CardAction`, `SelectGroup`, `SelectLabel`, `SelectScrollDownButton`, `SelectScrollUpButton`, `SelectSeparator`, `TableFooter`, `TableCaption` — exported by shadcn components but not all used in pages. These are standard shadcn patterns.

- **Unused file**: `src/components/ui/field.tsx` — installed by shadcn but not directly used by the app's form components.

- **Unused dependencies**: `@hookform/resolvers`, `react-hook-form` — installed by previous slice but not imported in current code. Retained for potential form validation needs.
