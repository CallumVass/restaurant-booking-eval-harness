# Restaurant Booking System

## Run backend

```bash
dotnet run --project src/RestaurantBooking.Api --urls http://localhost:5088
```
OpenAPI: `http://localhost:5088/openapi/v1.json`.

## Run frontend

```bash
npm --prefix frontend install
npm --prefix frontend run generate:client
npm --prefix frontend run build
npm --prefix frontend run typecheck
npm --prefix frontend run lint
npm --prefix frontend run format:check
npm --prefix frontend run deadcode
npm --prefix frontend run dev
```

The Vite dev server proxies `/restaurants` and `/bookings` to the backend. The typed React Query client is generated by Orval from `frontend/openapi.json`, captured from the backend OpenAPI document.

## Test/check backend

```bash
dotnet build src/RestaurantBooking.slnx -p:TreatWarningsAsErrors=true
dotnet test src/RestaurantBooking.slnx --no-build
dotnet format src/RestaurantBooking.slnx --verify-no-changes
```

## Notes

Persistence is in-memory for eval speed. Domain validation and table assignment stay in pure functions; API handlers are thin shells.
